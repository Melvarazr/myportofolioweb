PacilFlix: Database System Design for Streaming Platform

Academic team project to design a comprehensive database system for a streaming platform. Created complete Entity Relationship diagrams and relational schema to handle users, content management, subscriptions, and streaming analytics.

Overview

Worked as part of a 4-member team to design a database system for PacilFlix streaming platform. Created Enhanced Entity Relationship (EER) diagrams and mapped them to relational models. Designed main entities like USER, CONTENT, TRANSACTION, and EPISODE with their attributes and relationships. Used draw.io tools for database visualization and worked together with team members to create a complete database schema. Applied database concepts like entity relationships, primary keys, and foreign keys to build a working database design.

Details

Duration: February - March 2024 (2 months)

Team Size: 4 members

Course: Basis Data

Project Breakdown

1

Enhanced Entity Relationship (EER) Modeling

Created comprehensive EER diagrams to model the streaming platform's data structure. Designed entities for users, content management, subscriptions, and interactions. Implemented specialization/generalization concepts for content types (movies vs series) and established proper relationships between all system components.

Relational Database Schema Design

Mapped EER diagrams to relational database schema with proper normalization. Established primary and foreign key relationships, created junction tables for many-to-many relationships, and documented complete database structure with table specifications and constraints.
